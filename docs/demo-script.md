# GlowFit AI — 60초 데모 스크립트

이 스크립트는 [무음 화면 녹화](assets/glowfit-demo.webm)에 내레이션을 입히거나 면접에서 라이브 데모를 진행할 때 사용한다.

| 시간 | 화면 행동 | 말할 내용 |
| --- | --- | --- |
| 0–8초 | 추천 첫 화면에서 서비스 목적을 보여준다. | “GlowFit AI는 피부 조건에 맞는 제품을 추천하는 데서 멈추지 않고, 왜 이 제품이 추천됐는지 리뷰 근거까지 보여주는 스킨케어 추천 시스템입니다.” |
| 8–18초 | `건성 · 장벽 케어` 빠른 체험 프로필을 선택한다. | “직접 모든 조건을 입력할 수도 있지만, 포트폴리오 검토자가 핵심 흐름을 바로 볼 수 있도록 대표 프로필도 제공했습니다. 이 선택은 실제 추천 API 호출로 이어집니다.” |
| 18–32초 | 상위 3개와 1위 추천 이유, 태그, 점수/100, 리뷰 근거를 보여준다. | “결과에서는 단순한 정확도나 confidence를 말하지 않습니다. 입력 조건 적합도와, 제품 태그·리뷰 snippet으로 확인할 수 있는 근거를 분리해 보여줍니다.” |
| 32–42초 | `비교` 탭으로 이동한다. | “비교 화면에서는 후보별 적합도와 근거 강도, 주의 신호, 그리고 popularity·rating·content 같은 랭킹 신호를 같은 기준으로 볼 수 있습니다. 1.0 같은 정규화 값은 모델 성능이 아니라 이 후보 집합 안의 상대 신호입니다.” |
| 42–52초 | `리뷰 분석` 탭으로 이동한다. | “추천에 사용된 리뷰 문장을 aspect 단위로 노출해, 장벽 케어·건조한 피부·향 같은 근거가 실제 사용자 경험에서 왔는지 확인할 수 있게 했습니다.” |
| 52–60초 | `구성` 탭으로 이동한다. | “데이터는 Supabase 카탈로그에서, 추천은 FastAPI에서, 사용 경험은 Next.js에서 제공합니다. GitHub Actions가 배포 후 공개 웹과 API를 smoke check해 포트폴리오 데모의 운영 신뢰성도 확인합니다.” |

## 진행 팁

- 영상은 1280×800 데스크톱 뷰 기준으로 녹화했다. 실제 발표에서는 위 문장을 천천히 읽어 약 60초로 맞춘다.
- 제품 효능을 단정하지 말고, “리뷰에 나타난 사용자 경험”과 “현재 랭킹 신호”로 표현한다.
- 평가 질문이 나오면 현재 데이터셋은 탐색용이며, 시간 분할 user holdout과 더 큰 ASIN-joined 카탈로그가 다음 검증 단계라고 분명히 말한다.
